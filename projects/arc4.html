ARC4 Encryption Core

SystemVerilog DE1-SoC M10K RAM

Full ARC4 stream cipher in hardware, extended into a 51-core brute-force key cracker.

Overview

This project implements the complete ARC4 (alleged RC4) stream cipher as a hardware accelerator on the DE1-SoC FPGA. Five interdependent finite state machines coordinate through a custom ready-enable handshake protocol to share a single-port M10K memory. In a bonus extension, the design was scaled to 51 parallel cracking cores that search disjoint key ranges simultaneously, achieving ~93× speedup over a single core and exhausting all 224 candidate keys in roughly 1.5 seconds.

What I Built

  • init — Initializes the 256-byte S-box to identity permutation
  • ksa — Key Scheduling Algorithm; permutes S-box using the secret key
  • prga — Pseudo-Random Generation Algorithm; produces the cipher byte stream
  • arc4 — Top-level orchestrator sequencing Init → KSA → PRGA through ready-enable handshakes
  • crack — Single-core brute-force engine that iterates over a key range, running ARC4 decryption and validating output as printable ASCII
  • doublecrack — Parallel wrapper instantiating 51 crack cores with disjoint key partitions

Design Highlights

  • Ready-enable handshake protocol — Each FSM asserts rdy when idle and waits for en from its parent. This lets all five modules safely share a single-port M10K memory without bus arbitration hardware.
  • 51-core parallel cracking — The 224 key space is partitioned into 51 contiguous ranges. Each core runs independently with its own S-box memory, achieving ~93× wall-clock speedup (~1.5s total search).
  • ASCII validation for early termination — During PRGA, each decrypted byte is checked against the printable range (0x20–0x7E). A single non-printable character aborts that key immediately, avoiding wasted cycles on wrong keys.

Code: KSA State Machine

The Key Scheduling Algorithm permutes the S-box by swapping elements based on the secret key. Each state maps to a specific memory operation on the shared single-port RAM.

// ksa.sv — Key Scheduling Algorithm FSM
`define READY     3'b000
`define READ_MEM1 3'b001  // Read S[j]
`define R_W_MEM1  3'b010  // Write S[i] to address j
`define WRITE_MEM2 3'b011 // Write S[j] to address i
`define IDLE1     3'b100  // Set up read address for S[i]
`define IDLE2     3'b101  // Compute j = j + S[i] + key[i % 3]

always_comb begin
    case(state)
        `READY: begin
            if (en) next_state = `IDLE1;
            else    next_state = `READY;
        end
        `READ_MEM1: next_state = `R_W_MEM1;
        `R_W_MEM1:  next_state = `WRITE_MEM2;
        `WRITE_MEM2: begin
            if (I > 254) next_state = `READY;  // Done — all 256 entries permuted
            else         next_state = `IDLE1;   // Next iteration
        end
        `IDLE1: next_state = `IDLE2;
        `IDLE2: next_state = `READ_MEM1;
        default: next_state = `READY;
    endcase
end

// j update: the core KSA recurrence
`IDLE2: begin
    next_J = J + rddata + keys[I % KEYLENGTH];
end

Code: Crack Core — ASCII Validation

Each crack core runs ARC4 decryption and checks every output byte in real time. A single non-printable character flags the key as invalid, allowing immediate abort.

// crack.sv — Brute-force key search with early termination

// State transitions
always_comb begin
    case(state)
        READY: begin
            if (en) next_state = RUN_ARC4;
            else    next_state = READY;
        end
        RUN_ARC4: begin
            if (arc4_rdy && arc4_started) next_state = DONE_ARC4;
            else                          next_state = RUN_ARC4;
        end
        DONE_ARC4: begin
            if (test_key < dc_stopkey) begin
                if (pt_test == 1'b1) next_state = RUN_ARC4;  // Wrong key, try next
                else                 next_state = READY;      // Key found!
            end
            else next_state = READY;  // All keys exhausted
        end
        default: next_state = READY;
    endcase
end

// ASCII check during decryption — abort on non-printable byte
RUN_ARC4: begin
    if ((pt_wren == 1'b1) && (pt_addr != 8'd0)) begin
        if ((pt_wrdata < 8'h20) || (pt_wrdata > 8'h7E))
            next_pt_test = 1'b1;   // Non-printable → wrong key
        else
            next_pt_test = pt_test;
    end
end
